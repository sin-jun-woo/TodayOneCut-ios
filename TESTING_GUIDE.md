# TodayOneCut iOS - 테스트 가이드

## 📱 시뮬레이터에서 앱 실행하기

### 1. Xcode에서 프로젝트 열기
```bash
# 터미널에서
cd "/Users/shin-junwoo/Side Project/TodayOneCut-ios"
open TodayOneCut.xcodeproj
```

또는 Xcode에서 직접 `TodayOneCut.xcodeproj` 파일을 엽니다.

### 2. 시뮬레이터 선택
1. Xcode 상단 툴바에서 시뮬레이터 선택
   - 예: `iPhone 15 Pro` 또는 `iPhone 15 Pro Max`
2. iOS 버전: iOS 17.0 이상 권장

### 3. 빌드 및 실행
- **빌드**: `⌘B` (Command + B)
- **실행**: `⌘R` (Command + R)
- 또는 상단 툴바의 재생 버튼 클릭

### 4. 첫 실행 확인
- 온보딩 화면이 표시되어야 함
- "시작하기" 버튼 클릭 시 메인 화면으로 이동

---

## 🧪 기능별 테스트 시나리오

### 테스트 1: 온보딩 화면
**목표**: 첫 실행 시 온보딩이 정상적으로 표시되는지 확인

**절차**:
1. 앱 삭제 후 재설치 (시뮬레이터에서)
2. 앱 실행
3. 온보딩 화면 3페이지 확인
4. "시작하기" 버튼 클릭
5. 메인 화면(홈)으로 이동 확인

**예상 결과**:
- ✅ 온보딩 화면 3페이지 표시
- ✅ "시작하기" 버튼 클릭 시 메인 화면으로 이동
- ✅ 이후 앱 재실행 시 온보딩 화면 표시 안 됨

---

### 테스트 2: 기록 작성 (텍스트)
**목표**: 텍스트 기록 작성이 정상 동작하는지 확인

**절차**:
1. 홈 화면에서 "+" 버튼 클릭
2. "텍스트" 타입 카드 선택
3. 텍스트 입력 (예: "오늘의 테스트 기록")
4. 글자 수 표시 확인 (예: "15 / 500자")
5. "저장" 버튼 클릭
6. 홈 화면으로 돌아가서 기록 표시 확인

**예상 결과**:
- ✅ 기록 타입 선택 카드 표시
- ✅ 텍스트 입력 가능
- ✅ 글자 수 표시 (123 / 500자 형식)
- ✅ 저장 후 홈 화면에 기록 표시
- ✅ 기록 목록에 추가됨

---

### 테스트 3: 기록 작성 (사진)
**목표**: 사진 기록 작성이 정상 동작하는지 확인

**절차**:
1. 홈 화면에서 "+" 버튼 클릭
2. "사진" 타입 카드 선택
3. "갤러리에서 선택" 또는 "카메라로 촬영" 버튼 클릭
4. 사진 선택 또는 촬영
5. 선택한 사진이 표시되는지 확인
6. 메모 입력 (선택사항)
7. "저장" 버튼 클릭
8. 홈 화면으로 돌아가서 기록 표시 확인

**예상 결과**:
- ✅ 사진 선택/촬영 가능
- ✅ 선택한 사진 미리보기 표시
- ✅ 저장 후 홈 화면에 사진 기록 표시
- ✅ 기록 목록에 추가됨

---

### 테스트 4: 하루 1개 기록 제한
**목표**: 하루에 1개만 기록할 수 있는지 확인

**절차**:
1. 오늘 날짜로 기록 1개 작성 (테스트 2 또는 3 완료)
2. 다시 "+" 버튼 클릭하여 기록 작성 시도
3. 에러 메시지 확인

**예상 결과**:
- ✅ 두 번째 기록 작성 시도 시 에러 메시지 표시
- ✅ "하루에 하나의 기록만 작성할 수 있습니다" 같은 메시지
- ✅ 기록 작성 화면이 닫히지 않거나 적절한 피드백 제공

---

### 테스트 5: 기록 수정
**목표**: 당일 기록 수정이 정상 동작하는지 확인

**절차**:
1. 오늘 작성한 기록이 있는지 확인
2. 홈 화면에서 기록 카드 클릭 (또는 기록 목록에서)
3. 기록 상세 화면에서 "수정" 메뉴 클릭
4. 내용 수정 (텍스트 변경 또는 사진 변경)
5. "저장" 버튼 클릭
6. 수정된 내용 확인

**예상 결과**:
- ✅ 당일 기록 수정 가능
- ✅ 수정 후 변경사항 반영
- ✅ 수정 시간 업데이트

---

### 테스트 6: 기록 수정 제한 (1회)
**목표**: 당일 기록은 1회만 수정 가능한지 확인

**절차**:
1. 오늘 작성한 기록 수정 (테스트 5 완료)
2. 다시 수정 시도
3. 에러 메시지 확인

**예상 결과**:
- ✅ 두 번째 수정 시도 시 에러 메시지 표시
- ✅ "수정 횟수를 초과했습니다" 같은 메시지
- ✅ 저장 버튼 비활성화 또는 에러 표시

---

### 테스트 7: 과거 기록 수정 불가
**목표**: 과거 기록은 수정할 수 없는지 확인

**절차**:
1. 기록 목록에서 어제 또는 그 이전 날짜의 기록 선택
2. 기록 상세 화면 확인
3. "수정" 메뉴가 표시되지 않거나 비활성화되어 있는지 확인

**예상 결과**:
- ✅ 과거 기록의 "수정" 메뉴 비활성화 또는 숨김
- ✅ 수정 시도 시 "당일 기록만 수정할 수 있습니다" 에러

---

### 테스트 8: 기록 삭제
**목표**: 기록 삭제가 정상 동작하는지 확인

**절차**:
1. 기록 상세 화면으로 이동
2. "삭제" 메뉴 클릭
3. 삭제 확인 다이얼로그 확인
4. "삭제" 버튼 클릭
5. 기록 목록에서 삭제 확인

**예상 결과**:
- ✅ 삭제 확인 다이얼로그 표시
- ✅ 삭제 후 기록 목록에서 제거
- ✅ 홈 화면에서도 제거

---

### 테스트 9: 기록 목록 시간 역순
**목표**: 기록 목록이 최신순으로 표시되는지 확인

**절차**:
1. 기록 여러 개 작성 (날짜를 다르게)
2. 기록 목록 화면으로 이동
3. 목록 순서 확인

**예상 결과**:
- ✅ 최신 기록이 맨 위에 표시
- ✅ 오래된 기록이 아래에 표시

---

### 테스트 10: 검색 기능
**목표**: 기록 검색이 정상 동작하는지 확인

**절차**:
1. 기록 목록 화면으로 이동
2. 검색바에 키워드 입력 (예: "테스트")
3. 검색 결과 확인
4. 검색어 지우기
5. 전체 목록 표시 확인

**예상 결과**:
- ✅ 검색어 입력 시 관련 기록만 표시
- ✅ 검색어 지우기 시 전체 목록 표시
- ✅ 검색 결과가 시간 역순으로 정렬

---

### 테스트 11: 달력 화면
**목표**: 달력에서 기록 조회가 정상 동작하는지 확인

**절차**:
1. 달력 탭으로 이동
2. 기록이 있는 날짜 확인 (점 표시)
3. 기록이 있는 날짜 클릭
4. 해당 날짜의 기록 표시 확인
5. 월 이동 (이전/다음 월) 확인

**예상 결과**:
- ✅ 기록이 있는 날짜에 점 표시
- ✅ 날짜 클릭 시 해당 날짜의 기록 표시
- ✅ 일요일 빨간색, 토요일 파란색 표시
- ✅ 월 이동 정상 동작

---

### 테스트 12: 설정 화면
**목표**: 설정 변경이 정상 동작하는지 확인

**절차**:
1. 설정 탭으로 이동
2. 위치 정보 저장 토글 ON/OFF
3. 테마 모드 변경 (라이트/다크/시스템)
4. 앱 테마 변경
5. 폰트 선택 변경
6. 알림 설정 ON/OFF
7. 앱 재시작 후 설정 유지 확인

**예상 결과**:
- ✅ 각 설정 변경 즉시 반영
- ✅ 앱 재시작 후 설정 유지
- ✅ 테마 변경 시 UI 색상 변경
- ✅ 폰트 변경 시 텍스트 폰트 변경

---

5. 권한 거부 시 적절한 메시지 표시 확인

**예상 결과**:
- ✅ 권한 요청 다이얼로그 표시
- ✅ 권한 허용 시 기능 정상 동작
- ✅ 권한 거부 시 적절한 에러 메시지 표시
